🏥 MDTAgents: Codebase Overview

A Python multi-agent MDT (Multidisciplinary Team) orchestrator for medical case consultation. Drop any set of patient files into a case folder; AI agents automatically classify the documents, assemble the right specialist team, consult in parallel, and emit a structured final report — all without Python ever parsing medical content.

Level 1 — Entry Interfaces Level 2 — Workflow Orchestration Level 3 — Infrastructure

End-to-End Pipeline

Each run of python -m src.main <case_folder> executes five sequential rounds:

Round 0 🔍 Scanner 00_manifest.json
→
Round 1 🗂 Coordinator
Index 01_index.json
→
Round 2 📋 Coordinator
Dispatch 02_dispatch.json
→
Round 3 👥 Specialists
(parallel) 03_opinions/*.md
→
Round 4 📝 Coordinator
Synthesis 05_mdt_report.md

Workspace Artifacts (per Case)

Every case folder gets a .mdt_workspace/ sub-directory automatically. Agents communicate exclusively through these files — Python never parses medical content.

cases/demo_case/.mdt_workspace/ ├── 00_manifest.json # Scanner: file list + text previews + checksums ├── 01_index.json # Coordinator Round 1: file classification ├── 02_dispatch.json # Coordinator Round 2: specialist assignments ├── 03_opinions/ │ ├── 影像科.md # imaging specialist opinion │ ├── 病理科.md # pathology specialist opinion │ └── ... ├── 04_debate.json # (optional) cross-debate round ├── 05_mdt_report.md # final MDT consensus report └── errors/ └── {agent}.log # per-agent error logs

Code Organization — Three Layers

🚪 Layer 1 — Entry

  • src/main.py — CLI
  • app.py — Streamlit web UI

Accepts a case folder path, wires up all components, and drives the pipeline from start to finish.

🔄 Layer 2 — Orchestration

  • coordinator.py — 3-round engine
  • specialist_pool.py — parallel pool

Coordinator issues LLM Agent calls for index → dispatch → synthesis. Specialist pool runs each specialist in its own thread.

🔧 Layer 3 — Infrastructure

  • scanner.py — file scanner
  • file_bus.py — workspace I/O bus
  • cli_client.py — OpenCode wrapper

Handles all I/O: extracting text from files, managing workspace paths, and invoking the OpenCode CLI subprocess.

Quick Start Example

Install & Run (CLI)

# 1. Install Python dependencies
pip install -r requirements.txt

# 2. Ensure OpenCode CLI is on your PATH
opencode --version

# 3. Run MDT on the demo case
python -m src.main cases/demo_case

# 4. Read the final report
cat cases/demo_case/.mdt_workspace/05_mdt_report.md

Run the Web UI

pip install -r requirements-ui.txt
streamlit run app.py

The web UI provides three tabs: 🏥 Run MDT (upload or select a case, run pipeline, download report), 🔍 Debug (browse all intermediate workspace JSON/MD files), and ⚙️ Admin (edit model settings and specialist registry).

Configuration Example (config/system.yaml)

opencode:
  default_model: "claude-sonnet-4"   # LLM model used by all agents
  timeout: 300                        # seconds before an agent call is killed
  max_workers: 5                      # maximum parallel specialists

specialists:
  - name: "影像科"                    # must match prompts/specialists/影像科.md
    model: "claude-sonnet-4"
    file_categories: ["影像"]
  - name: "病理科"
    model: "claude-sonnet-4"
    file_categories: ["病理"]
  - name: "肿瘤内科"
    model: "claude-sonnet-4"
    file_categories: ["检验", "病历", "既往治疗"]

workflow:
  enable_debate: false                # set true to enable Round 4 cross-debate
