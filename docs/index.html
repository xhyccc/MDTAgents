MDTAgents: Codebase Overview

This project is a Python-based multi-agent MDT (medical team discussion) orchestrator. It scans case files, asks LLM agents to classify and dispatch specialists, runs specialists in parallel, and writes a final report into a workspace folder.

API Level 1 (Entry Interfaces) API Level 2 (Workflow Orchestration) API Level 3 (Infrastructure)

Key Technologies

  • Language: Python 3.10+
  • LLM Runtime: OpenCode CLI (opencode run)
  • UI: Streamlit (app.py)
  • Config: YAML (config/system.yaml)
  • Testing: Pytest (unit + integration)
  • File parsing: pdfplumber, python-docx, openpyxl

Top-Level Structure

  • src/: Core pipeline modules (scanner, coordinator, specialist pool, file bus, CLI client, entry point)
  • prompts/: Coordinator and specialist prompt templates
  • config/system.yaml: Runtime/model/concurrency/specialist registry configuration
  • cases/: Input case folders (sample provided in cases/demo_case)
  • tests/: Unit and integration tests for the workflow
  • app.py: Streamlit web interface

How Code Is Organized

  1. Entry layer: CLI and web UI trigger pipeline execution.
  2. Orchestration layer: Coordinator and specialist pool execute multi-round workflow.
  3. Infrastructure layer: Scanner, file-bus, and OpenCode wrapper handle I/O and execution plumbing.

Pipeline Artifacts

Each case gets a workspace at .mdt_workspace/ containing:

  • 00_manifest.json (scanner output)
  • 01_index.json (coordinator round 1)
  • 02_dispatch.json (coordinator round 2)
  • 03_opinions/*.md (specialist outputs)
  • 05_mdt_report.md (final synthesis)
  • errors/*.log (agent failures)
