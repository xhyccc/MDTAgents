API Level 2: Workflow Orchestration

Two classes drive the MDT workflow: Coordinator (sequential, 3-round) and SpecialistPool (parallel). Together they implement the complete pipeline from file classification to final report.

Coordinator (src/coordinator.py)

The Coordinator is the control-plane agent. It has three public methods, one per round. Each method renders a prompt template, calls the LLM via OpenCodeClient, parses the JSON response, and persists the result via FileBus.

API Summary

MethodInputOutput (file)Agent role
run_index(manifest) Manifest object 01_index.json Classifies each file into a medical category and assesses completeness
run_dispatch(index) index dict 02_dispatch.json Decides which specialist departments are needed and assigns files to each
run_synthesis(index, opinions) index dict + opinions dict 05_mdt_report.md Synthesises all specialist opinions into a final MDT consensus report

Three-Round Flow

Round 1
Index
Coordinator: file classification
Reads 00_manifest.json → asks agent to label each file (e.g. "影像", "病理", "检验") and flag missing data → writes 01_index.json
↓
Round 2
Dispatch
Coordinator: specialist team assembly
Reads 01_index.json + available prompt files → asks agent which specialists to involve and which files each should read → writes 02_dispatch.json
↓ (after SpecialistPool)
Round 4
Synthesis
Coordinator: consensus report
Reads all 03_opinions/*.md → asks agent (as MDT chairperson) to merge findings, identify consensus and disagreements → writes 05_mdt_report.md

Constructor

from pathlib import Path
from src.file_bus import FileBus
from src.coordinator import Coordinator

bus = FileBus(Path("cases/demo_case"))
bus.init_workspace()

coordinator = Coordinator(
    bus=bus,
    config_path=Path("config/system.yaml"),
    prompts_dir=Path("prompts"),
)

# Round 1
index = coordinator.run_index(manifest)

# Round 2
dispatch = coordinator.run_dispatch(index)

# Round 4 (after specialist pool)
coordinator.run_synthesis(index, opinions)

Specialist Pool (src/specialist_pool.py)

SpecialistPool reads specialists_required from the dispatch plan and launches one OpenCode session per specialist in a ThreadPoolExecutor. Individual specialist failures are caught and logged without blocking the others.

API

from src.specialist_pool import SpecialistPool

pool = SpecialistPool(bus, config_path=config_path, prompts_dir=prompts_dir)

# Returns {specialist_name: opinion_text} for all completed specialists
opinions: dict[str, str] = pool.run_parallel(dispatch)

Per-Specialist System Prompt Assembly

# _build_system_prompt() concatenates two prompt files:
#   prompts/specialists/base.md        ← shared clinical rules
#   prompts/specialists/{name}.md      ← specialty-specific instructions
# Result is joined with "---" separator and sent as the system turn.

Example Specialist Opinion (03_opinions/影像科.md)

# 影像科会诊意见

## 一、资料概述
阅读 CT检查.md：腹部CT平扫及增强扫描报告。

## 二、专科分析
肝右叶见一约 4.2cm × 3.8cm 低密度占位，增强后动脉期明显强化，
门脉期及延迟期廓清，符合肝细胞癌（HCC）典型影像表现。
门静脉主干未见明显癌栓。

## 三、初步结论
影像学表现符合原发性肝细胞癌，BCLC 分期初步为 B 期。

## 四、需要补充的资料
建议行肝脏 MRI 平扫 + 增强以进一步明确病灶边界。
