API Level 1: Entry Interfaces

There are two ways to trigger the MDT pipeline: a CLI and a Streamlit web UI. Both delegate all execution to python -m src.main; neither parses medical content.

1 — CLI Entry (src/main.py)

Signature

python -m src.main <case_folder>

What it does (step by step)

  1. Resolves <case_folder> to an absolute path and validates it exists.
  2. Locates config/system.yaml and prompts/ relative to the package root.
  3. Round 0 — creates .mdt_workspace/ via FileBus.init_workspace() and runs Scanner.scan().
  4. Rounds 1–2 — calls Coordinator.run_index() then Coordinator.run_dispatch().
  5. Round 3 — calls SpecialistPool.run_parallel() (threaded).
  6. Round 4 — calls Coordinator.run_synthesis().
  7. Prints the report path and workspace path on success.

Complete Example

# Run MDT on the bundled demo case
python -m src.main cases/demo_case

# Run on a custom case folder
python -m src.main /data/patients/case_001

# After completion, read the report
cat cases/demo_case/.mdt_workspace/05_mdt_report.md

Web-to-CLI Delegation

# app.py — how the web UI executes the pipeline
def _run_mdt_subprocess(case_dir: Path) -> Tuple[int, str, str]:
    """Run `python -m src.main <case_dir>` synchronously."""
    result = subprocess.run(
        [sys.executable, "-m", "src.main", str(case_dir)],
        capture_output=True,
        text=True,
        encoding="utf-8",
        cwd=str(PROJECT_ROOT),
    )
    return result.returncode, result.stdout, result.stderr

The web UI never touches the pipeline internals directly — it only spawns the CLI subprocess and displays its output.
