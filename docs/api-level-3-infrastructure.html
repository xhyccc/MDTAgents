API Level 3: Infrastructure & Adapters

Three modules form the infrastructure layer: scanner.py extracts text from files, file_bus.py manages the workspace as an inter-agent message bus, and cli_client.py wraps the OpenCode CLI subprocess.

Scanner (src/scanner.py)

Recursively scans a case directory and returns a Manifest containing a text preview and metadata for every supported file. No medical keywords or type detection — all classification is left to the Coordinator Agent.

API

from pathlib import Path
from src.scanner import Scanner

scanner = Scanner()
manifest = scanner.scan(Path("cases/demo_case"))

print(manifest.total_files)       # → 4
print(manifest.files[0].path)     # → "CT检查.md"
print(manifest.files[0].preview)  # → first 800 chars of the file
print(manifest.files[0].checksum) # → MD5 hex digest

Data Models

FileEntry (dataclass)

path str relative path from case_dir
size int file size in bytes
mime_type str guessed MIME type
preview str first 800 chars of text content
checksum str MD5 hex digest

Manifest (dataclass)

case_id str name of case folder
files List[FileEntry] all scanned files
total_files int count of files
timestamp str ISO 8601 UTC scan time

File Bus (src/file_bus.py)

FileBus is the inter-agent message bus. All communication between the Coordinator, specialist agents, and the Python orchestrator happens by writing and reading files under .mdt_workspace/. Python never parses medical content.

Workspace Directory Layout

cases/demo_case/.mdt_workspace/ ├── 00_manifest.json ← FileBus.save_manifest() / load_manifest() ├── 01_index.json ← FileBus.save_index() / load_index() ├── 02_dispatch.json ← FileBus.save_dispatch() / load_dispatch() ├── 03_opinions/ │ ├── 影像科.md ← FileBus.save_opinion("影像科", text) │ └── ... ← FileBus.load_opinions() → dict ├── 04_debate.json ← FileBus.save_debate() (optional) ├── 05_mdt_report.md ← FileBus.save_report() └── errors/ └── {agent}.log ← OpenCodeClient._write_error()

API Reference

MethodDescription
init_workspace()Creates workspace, opinions, and errors directories (idempotent)
save_manifest(manifest)Serialise and write Manifest as JSON
save_index(index)Write index dict as 01_index.json
save_dispatch(dispatch)Write dispatch dict as 02_dispatch.json
save_opinion(name, text)Write 03_opinions/{name}.md
save_report(text)Write 05_mdt_report.md
load_opinions()Returns {name: text} for all *.md files in 03_opinions/

Usage Example

from pathlib import Path
from src.file_bus import FileBus

bus = FileBus(Path("cases/demo_case"))
bus.init_workspace()

# Write intermediate results
bus.save_manifest(manifest)
bus.save_index(index_dict)
bus.save_dispatch(dispatch_dict)
bus.save_opinion("影像科", "# 影像科会诊意见\n...")

# Read all specialist opinions
opinions = bus.load_opinions()
# → {"影像科": "# 影像科...", "病理科": "# 病理科..."}

print(bus.report_path)   # → .mdt_workspace/05_mdt_report.md
print(bus.workspace_dir) # → .mdt_workspace/

OpenCode Adapter (src/cli_client.py)

OpenCodeClient wraps every LLM Agent invocation as an opencode run subprocess call. The system prompt is written to a temporary file (to avoid shell length limits), the user message and optional file attachments are passed as flags, and stdout is returned as the Agent's response.

CLI Call Flow

Python caller
(Coordinator / SpecialistPool)
→
client.run(…)
write system prompt
to temp file
→
opencode run
--model …
--system-prompt …
--message …
--file …
→
stdout
(Agent response text)

Generated Shell Command (example)

opencode run \
  --model claude-sonnet-4 \
  --system-prompt /tmp/tmpabc123.md \
  --message "你是影像科专家。请阅读下方病例资料…" \
  --file cases/demo_case/CT检查.md

API

from pathlib import Path
from src.cli_client import OpenCodeClient, AgentError

client = OpenCodeClient(
    error_log_dir=Path(".mdt_workspace/errors"),
    default_model="claude-sonnet-4",
)

try:
    response: str = client.run(
        agent_name="coordinator_index",
        system_prompt="You are the MDT coordinator …",
        user_message="Classify these files: …",
        file_paths=[Path("cases/demo_case/CT检查.md")],
        model="claude-sonnet-4",
        timeout=300,
    )
except AgentError as e:
    # Error already written to .mdt_workspace/errors/coordinator_index.log
    print(f"Agent failed: {e}")

Error Handling

ConditionExceptionSide effect
Process exceeds timeoutAgentErrorWrites errors/{agent}.log
opencode not on PATHAgentErrorWrites errors/{agent}.log
Non-zero exit codeAgentErrorWrites stderr to errors/{agent}.log
Specialist failsCaught in pool, loggedOther specialists continue unaffected
Coordinator failsPropagates to main.pyPipeline terminates
